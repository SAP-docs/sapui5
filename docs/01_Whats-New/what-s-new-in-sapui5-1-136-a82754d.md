**Removal of *Belize* and *Blue Crystal* Themes**

We have removed the deprecated *Belize* theme family \(theme IDs: `sap_belize`, `sap_belize_plus`, `sap_belize_hcb`, `sap_belize_hcw`\). We have also removed the *Blue Crystal* theme and the related *High Contrast Black* theme \(theme IDs: `sap_bluecrystal`, `sap_hcb`\), which have been deprecated for some time. Themes that belong to the removed theme families \(including custom themes based on these themes\) will automatically be switched to the *Horizon* theme family in your application \(theme IDs: `sap_horizon`, `sap_horizon_dark`, `sap_horizon_hcb`, `sap_horizon_hcw`\).

**SAPUI5 OData V4 Model** 

</td>
<td valign="top">

**SAPUI5 OData V4 Model**

The new version of the SAPUI5 OData V4 model introduces the following features:

-   The selection validation introduced with SAPUI5 1.135 now also works for recursive hierarchies, that is, if the `hierarchyQualifier` is set in `sap.ui.model.odata.v4.ODataListbinding#setAggregation` or in the `$$aggregation` binding parameter.

    When reloading a list binding's data, the selection state is now validated if the `$$clearSelectionOnFilter` list binding parameter is set and if either `$$aggregation` is not set or the `hierarchyQualifier` is set. The selection state of contexts no longer matching the filter is reset.For more information, see the API Reference for [`v4.ODataListbinding#setAggregation`](https://ui5.sap.com/#/api/sap.ui.model.odata.v4.ODataListbinding%23methods/setAggregation) and [`v4.ODataModel#bindList`](https://ui5.sap.com/#/api/sap.ui.model.odata.v4.ODataModel%23methods/bindList).

-   The `Promise` returned by `sap.ui.model.odata.v4.Context#move` now resolves with an index if the experimental `copy` parameter is set. For more information, see the [API Reference](https://ui5.sap.com/#/api/sap.ui.model.odata.v4.Context%23methods/move).

**SAP Fiori Elements for OData V4**

The following changes and new features are available for SAP Fiori elements for OData V4:

-   You can now use `aggregationOnLeafLevel` to display aggregation based solely on visible columns in an analytical table. For more information, see [Setting the Table Type](../06_SAP_Fiori_Elements/setting-the-table-type-7f844f1.md).

-   Tree tables now support the copy and paste functionality. For more information, see [Tree Tables](../06_SAP_Fiori_Elements/tree-tables-7cf7a31.md).

-   We now support inline edit in the list report and on the object page. For more information, see [Inline Edit](../06_SAP_Fiori_Elements/inline-edit-bb56175.md).

-   We've renamed the API of the `Field` building block from `FieldAPI` to `Field`. For information about the `Field` API, see the [API Reference](https://ui5.sap.com/#/api/sap.fe.macros.Field).

-   You can now use the `Field` building block to dynamically create fields at runtime. You can use this building block within controllers for more flexibility and a wider variety of use cases. For more information, see [The Field Building Block](../06_SAP_Fiori_Elements/the-field-building-block-5260b9c.md).

-   You can now configure the `semanticObject` and `readOnly` properties in the `manifest.json` file for the `Field` building block. For more information, see [The Field Building Block](../06_SAP_Fiori_Elements/the-field-building-block-5260b9c.md).

-   You can now choose to not persist personalization changes applied to the `FilterBar` building block. For more information, see [The FilterBar Building Block](../06_SAP_Fiori_Elements/the-filterbar-building-block-7838611.md).

-   We've introduced a `Page` building block that allows freestyle applications to create custom pages with a layout similar to the object page. For more information, see [The Page Building Block](../06_SAP_Fiori_Elements/the-page-building-block-0258650.md).

-   You can now use the `MessageButton` building block. Note that this building block is an experimental feature and subject to revisions. For more information, see [The MessageButton Building Block](../06_SAP_Fiori_Elements/the-messagebutton-building-block-b365f2a.md).

-   If a user tries to access a collaborative draft but isn't authorized to modify draft data of the business object, we now display a warning, and the draft is opened in display mode. For more information, see [Draft Handling](../06_SAP_Fiori_Elements/draft-handling-ed9aa41.md).

-   You can now add an icon to indicate that a button is related to an AI action. For more information, see [Adding the AI Icon to Action Buttons](../06_SAP_Fiori_Elements/adding-the-ai-icon-to-action-buttons-3ad0452.md).

-   We've introduced a toggle button on the column selection tab in the table's `p13N` dialog, to allow end users to display description columns when needed. For more information, see [Enabling Table Personalization](../06_SAP_Fiori_Elements/enabling-table-personalization-3e2b4d2.md).

**`sap.m.upload.FilePreviewDialog`**

We've now introduced the `customPageContentHandler` property within `sap.m.upload.FilePreviewDialog`. The callback property allows developers to provide a custom control to preview unsupported file types in the preview dialog of the `sap.m.plugin.UploadSetwithTablePlugin`.

This property as a callback function returns a promise. Developers can resolve the promise with a control, for example, the `sap.ui.codeeditor.CodeEditor` for XML files. If the promise is rejected, the `sap.m.plugin.UploadSetwithTablePlugin` renders a default illustrated message in the file preview.

For more information, see [API Reference](https://ui5.sap.com/#/api/sap.m.upload.FilePreviewDialog) and the [Sample](https://ui5.sap.com/#/entity/sap.m.plugins.UploadSetwithTable/sample/sap.m.sample.UploadSetwithTablePlugin.FilePreviewCustomPageContent).
