**`sap.tnt.NavigationListItem`**

You can now add tags to navigation list items using the new `tag` aggregation. Tags use `sap.m.ObjectStatus` to display status indicators, counters, and metadata, such as *Beta*, *New*, *5 Pending*, or *Critical*, directly on navigation entries. Tags are visible when the navigation list is expanded and hidden when collapsed. For more information, see the [API Reference](https://ui5.sap.com/#/api/sap.tnt.NavigationListItem) and the [Samples](https://ui5.sap.com/#/entity/sap.tnt.NavigationListItem).

**`sap.suite.ui.microchart.HarveyBallMicroChart`** 

</td>
<td valign="top">

**`sap.suite.ui.microchart.HarveyBallMicroChart`**

We've introduced a new Boolean property `xsWidth` to the Harvey Ball micro chart. The default value is set to `false`. When you set the property to `true`, the control applies the following revised dimensions for the XS size variant:

-   Width increases from 6 rem to 7 rem.

-   Font size of the value label decreases from 0.875 rem to 0.75 rem.

-   Height remains 1.5 rem \(table row height is not affected\).


This property addresses clipping issues in tables where preformatted numeric labels exceeded the 6 rem container width.

For more information, see the [API Reference](https://ui5.sap.com/#/api/sap.suite.ui.microchart.HarveyBallMicroChart) and the [Sample](https://ui5.sap.com/#/entity/sap.suite.ui.microchart.HarveyBallMicroChart).

<td valign="top">

**SAP Fiori Elements for OData V4**

The following changes and new features are available for SAP Fiori Elements for OData V4:

-   Navigating from object pages in edit mode no longer triggers confirmation popups for discarding drafts when users made no changes. We recommend that you adjust any tests that expect a different behavior. For more information, see [Draft Handling](../06_SAP_Fiori_Elements/draft-handling-ed9aa41.md).

-   You can now display and remove a custom message strip on a list report page. The message strip supports formatting and can include links. For more information, see [Adding a Custom Message Strip to List Report Page, Analytical List Page and Object Page](../06_SAP_Fiori_Elements/adding-a-custom-message-strip-to-list-report-page-analytical-list-page-and-object-page-0fdbefb.md).

-   You can now define `formatOptions` for all fields of an annotation-based column. For more information, see [The Table Building Block](../06_SAP_Fiori_Elements/the-table-building-block-3801656.md).

-   You can now use semantic date operators such as `"Today"` or `"Last Week"` in the filter bar of overview pages to filter data across multiple cards. For more information, see [Enabling Semantic Operators in the Filter Bar](../06_SAP_Fiori_Elements/enabling-semantic-operators-in-the-filter-bar-fef65d0.md).

**`sap.ui.model.odata.type.ODataType`** 

</td>
<td valign="top">

**`sap.ui.model.odata.type.ODataType`**

The `sap.ui.model.odata.type.ODataType#getPlaceholderText` method now accepts two new optional parameters, `oMinimum` and `oMaximum`. You can use them to compute a placeholder that fits within a specific date range. This feature is available for all OData types that use `sap.ui.core.format.DateFormat` for formatting:

-   `sap.ui.model.odata.type.Date`
-   `sap.ui.model.odata.type.DateTime`
-   `sap.ui.model.odata.type.DateTimeOffset`
-   `sap.ui.model.odata.type.DateTimeWithTimezone`
-   `sap.ui.model.odata.type.Time`
-   `sap.ui.model.odata.type.TimeOfDay`

For more information, see the [API Reference](https://ui5.sap.com/#/api/sap.ui.model.odata.type.ODataType%23methods/getPlaceholderText).

**SAPUI5 OData V4 Model**

<td valign="top">

**SAPUI5 OData V4 Model**

The new version of the SAPUI5 OData V4 model introduces experimental support for the following features when using data aggregation without group levels:

-   When a side effect changes a property, the header context is marked as outdated only if that property is used in a `Sorter` or `Filter` object, or if a `$search` or custom query option is set. The grand total's context is marked as outdated if the changed property is used in a `Filter` object, or if a `$search` or custom query option is set.

-   The `groupPaths` provided with an `sap.ui.model.Sorter` are evaluated when you apply these sorters using `sap.ui.model.odata.v4.ODataListBinding#sort`. The binding adds these paths to the `$select` and `$expand` query options if you use auto-$expand/$select. For more information, see the [API Reference](https://ui5.sap.com/#/api/sap.ui.model.odata.v4.ODataListBinding%23methods/sort).
