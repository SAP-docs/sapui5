# SAP Fiori Elements for OData V4

Develop apps using SAP Fiori elements for OData V4 and benefit from the convenience of building blocks, the structure of standard floorplans, and freedom to adapt your app.

To generate the UI, SAP Fiori elements for OData V4 modularizes visible UI patterns and application behavior into building blocks: reusable UI components and interaction patterns that provide the framework benefits. Building blocks are pre-built and automatically comply with [SAP Design System](https://www.sap.com/design-system/fiori-design-web/) standards from the framework.

You can consume the building blocks in standard floorplans or in custom pages, or you can combine both approaches to best fit your application requirements.

-   [Standard floorplans](standard-floorplans-797c323.md) are generated based on metadata. They combine the most common building blocks and patterns in predefined layouts with added optimization for component interaction at the floorplan level. Using standard floorplans is the fastest and most efficient way to create SAP Fiori apps. You can extend your app for specific deviations by using building blocks or custom SAPUI5 coding.


-   [Custom pages](custom-page-0210497.md) provide even more flexibility. You can combine building blocks with additional SAPUI5 coding to achieve custom layouts while still benefiting from the framework capabilities.


Visit [SAP Fiori Development Portal](https://ui5.sap.com/test-resources/sap/fe/core/fpmExplorer/index.html#/topic/introduction) to explore framework features and their adaptation options on the front end and back end in real time.

Search, filter, and browse the detailed list of features in the [SAP Fiori Elements Feature Map](sap-fiori-elements-feature-map-62d3f7c.md).
